Making a canvas

In the previous exercise we used a list of lists to create a two dimensional grid, which we printed to create a crude diagram on the screen. This exercise takes these ideas a little further. The code below creates and displays a "canvas" of arbitrary length and width. The "." character is placed in every cell in the canvas so we can easily verify its dimensions. 
def create_canvas(x, y, char):
    canvas = []

    for i in range(x):
        row = [char for j in range(y)]

        canvas.append(row)

    return canvas

def display_canvas(canvas):
    for row in canvas:
        print("".join(row))

canvas = create_canvas(10, 25, ".")

display_canvas(canvas)

Aside: the usual convention in maths, which we learn in primary school, is that the x-axis is horizontal, and the y-axis is vertical. However, due to the way in which these pictures are being printed in text, our graphs are rotated, clockwise, by 90 degrees: the x-axis has become the vertical axis, and the y-axis as become the horizontal axis. While this isn't the way most of us have been trained to think about coordinates, it does not break anything. Our coordinate system is perfectly valid and consistent within itself, and the more you work with it the more you will get used to it. We can "draw" on the canvas simply by assigning single characters to the cells in the two dimensional list of lists. For example, the code below shows how the X-shape given in the example in the previous grid exercise can be rendered in code. 
canvas = create_canvas(7, 7, ".")

canvas[2][2] = "*"
canvas[2][4] = "*"
canvas[3][3] = "*"
canvas[4][2] = "*"
canvas[4][4] = "*"

display_canvas(canvas)
Here's the output from the above code.
